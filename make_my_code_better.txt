1) Жёсткие нарушения PEP8
Просто, давайте следовать требованиям PEP8
2) Нейминг
Именование - это важно, правильно называй переменные: прочитал название - понял что делает функция или переменная
3) Комментарии
Комментарии не должны пояснять что делает код, а зачем этот код написан
4) Документация по проекту
Точнее её отсутствие ;)
5) Бизнес-логика в контроллерах
Работа с БД или слишком большая view'шка - значит бизнес-логика
6) Использование функций
Функции не должны быть слишком длинными
7) Изучите ваш фреймворк
прежде чем делать это самому - посмотри не сделано ли это в твоём фреймворке
8) Исключения и обработка ошибок
нет обработки - плохо
9) Логирование
нет логирования - плохо
10) Конфиденциальные данные в коде и git репозиториях
Не вставляй API ключи в код! Не храни ключи в репозитории, ключи - в переменных окружения
11) pip requirements и состав git репозитория
Нет файла где описаные все пакеты которые нужно установить для проекта
12) Преждевременные попытки внедрить паттерны
Не усложняй, keep it stupid simple!
